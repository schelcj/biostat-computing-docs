* A Compute-intensive task is defined as any process that dominates CPU cycles (e.g. uses more than 10% CPU-utilization or pushes the machine load above 1.0) or memory usage on the machine. For example, any SAS, Splus or Matlab process is considered compute-intensive. All statistical programs and simulations are considered compute-intensive. An SPH user's fortran program is likely to be compute-intensive. Running Firefox or another graphical web browser is sometimes a compute-intensive task.
